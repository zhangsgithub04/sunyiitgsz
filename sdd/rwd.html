Fluid Grids & Flexbox

Gone are the days of fixed-pixel widths. Responsive design uses relative units (%, vw, vh) and modern CSS layout engines like Grid and Flexbox to redistribute space dynamically.

container { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); }

Media Queries: The Breakpoint Strategy

Instead of designing for "iPhone" or "iPad," we design for content breakpoints. We start Mobile First and add complexity as the viewport expands.

Mobile (< 640px)

Whiteboard tools collapse into a bottom sheet; canvas fills 100% of the screen.

Tablet (< 1024px)

Sidebars become toggleable drawers; typography scales up for better legibility.

Desktop (> 1024px)

Permanent left toolbar, right participant list, and keyboard shortcut overlays.

Flexible Media & SVGs

For our whiteboard, bitmaps (JPG/PNG) can become pixelated when zoomed. We prioritize SVG (Scalable Vector Graphics) for all strokes and shapes because they remain crisp at any resolution or screen size.

Implementation Tip:

Use the srcset attribute for raster images to serve small files to phones and high-res files to Retina displays, saving bandwidth for mobile collaborators.

Touch Targets & Fitt's Law

Desktop users have pixel-perfect precision with a mouse. Mobile users have "fat fingers." RWD best practice dictates that touch targets (buttons, links) should be at least 44x44 points.

  • Gestures: Implementing pinch-to-zoom and two-finger pan for tablet users.
