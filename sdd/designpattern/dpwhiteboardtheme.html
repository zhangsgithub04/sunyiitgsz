Mastering UML Diagrams

Learning the 14 UML standards through the lens of building a Real-time Collaborative Whiteboard Application.

Structural Diagrams (7)

Structural diagrams define the static architecture of the system—the "blueprints" of what exists within the whiteboard app.

1. Class Diagram

Shows system classes, attributes, and relationships.

Whiteboard Scenario

Defining the properties of a CanvasElement class and its subclasses like StickyNote, Shape, and Arrow.

2. Object Diagram

A snapshot of instances at a specific moment.

Whiteboard Scenario

Showing a specific StickyNote (ID: 101, Color: Yellow) currently placed on "Project Alpha" Board.

3. Package Diagram

Organizes elements into logical groups.

Whiteboard Scenario

Grouping the code into UI Components, Real-time Engine (WebSockets), and Storage Layer.

4. Component Diagram

Shows high-level modular parts of the system.

Whiteboard Scenario

Visualizing how the Image Processing Service interacts with the S3 Bucket Component via an API.

5. Composite Structure Diagram

Details the internal structure of a class/classifier.

Whiteboard Scenario

Breaking down the Toolbar class to show how the ColorPicker and BrushSize ports connect internally.

6. Deployment Diagram

Shows physical hardware and software execution environments.

Whiteboard Scenario

Mapping the Node.js Server onto an AWS EC2 instance and the React App onto the User's Browser.

7. Profile Diagram

Extends UML for specific domains using stereotypes.

Whiteboard Scenario

Creating a custom «RealTime» stereotype to apply to classes that require sub-50ms latency.

Behavioral Diagrams (7)

Behavioral diagrams describe what happens in the system—the dynamic movements and interactions between users and data.

1. Use Case Diagram

Summarizes user goals and system boundaries.

Whiteboard Scenario

A Designer (Actor) needs to Create Board, Invite Collaborator, and Export to PDF.

2. Activity Diagram

Models the workflow or business process.

Whiteboard Scenario

The step-by-step logic of User Login: Enter Email -> Auth Check -> [Success] Load Dashboard / [Fail] Show Error.

3. State Machine Diagram

Shows the lifecycle of an object via states and transitions.

Whiteboard Scenario

A Connection object transitioning from Connecting to Active, Reconnecting, or Disconnected.

4. Sequence Diagram

Focuses on the time-ordered exchange of messages.

Whiteboard Scenario

User A moves a square -> Frontend sends 'update' to Server -> Server broadcasts 'sync' to User B -> User B's screen updates.

5. Communication Diagram

Focuses on organizational relationships between objects.

Whiteboard Scenario

Visualizing how the WhiteboardManager talks to 5 different UserSession objects simultaneously.

6. Interaction Overview Diagram

A high-level mix of Activity and Sequence diagrams.

Whiteboard Scenario

An overview showing the choice between Join via Link vs Join via Email, containing smaller sequence flows for each.

7. Timing Diagram

Focuses on timing constraints and state changes over time.

Whiteboard Scenario

Ensuring the Auto-Save trigger happens exactly 2 seconds after the last StrokeEvent to prevent server overload.
